# LadderSFC_Step Framework Documentation

## Overview
The LadderSFC_Step framework is designed to provide a structured approach to implementing sequential function charts (SFC) in industrial automation systems. This documentation will cover the architecture, data types, AOI (Add-On Instruction) components, parameters, and typical usage patterns of the framework.

## Architecture
The architecture of the LadderSFC_Step framework is based on modular components, enhancing reusability and maintainability. Key components include:
1. **Main Execution Loop**: Coordinates the execution of SFC steps on a cyclic basis.
2. **Step Components**: Individual steps that represent specific operations or actions within the process.
3. **Transition Conditions**: Conditions that trigger the transition between steps.
4. **Data Management**: Structures for managing and sharing data among steps.

## Data Types
The framework utilizes several data types to manage state and configuration:
- **BOOL**: Represents binary states (e.g., true/false).
- **INT**: Represents integer values for counters and indexes.
- **REAL**: Represents floating-point values used for measurements and setpoints.
- **STRING**: Represents text values for logging or display purposes.
- **ARRAY**: Collections of values for managing multiple state variables or configurations.

## AOI Components
The framework includes multiple AOI components that encapsulate the functionality of different steps:
1. **Step_AOI**: Represents a single step in the SFC with inputs, outputs, and logic.
2. **Transition_AOI**: Represents the conditions and logic that determine when to transition between steps.
3. **State_Machine_AOI**: Manages the overall state of the SFC and transitions between groups of steps.

## Parameters
Each AOI component has configurable parameters to allow for flexibility in usage:
- **Input Parameters**: Conditions and criteria to enter the step or transition.
- **Output Parameters**: Actions or outputs generated by the step.
- **Configuration Parameters**: Setpoints and operational parameters for customizing behavior.

## Usage Patterns
Common usage patterns include:
1. **Sequential Execution**: Define a series of steps with transitions based on binary conditions.
2. **Parallel Execution**: Manage multiple branches of execution within a single SFC.
3. **Event-Driven Execution**: Trigger steps based on external events or conditions that may occur outside the regular cycle.

## Conclusion
The LadderSFC_Step framework provides a powerful tool for implementing complex automation systems effectively. By utilizing its modular structure, developers can create reusable and maintainable code, resulting in more efficient industrial applications.

## Example Code Snippet
```ladder
// Example of a simple SFC with two steps
Step_AOI Step1(
    Input:= Condition1,
    Output:= Action1
);

Step_AOI Step2(
    Input:= Action1,
    Output:= Condition2
);

Transition_AOI Transition1(
    Condition:= Condition2
);
```
